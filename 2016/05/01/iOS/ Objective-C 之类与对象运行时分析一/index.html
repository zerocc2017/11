基类 NSObject 只有一个 Class 类型的 isa 成员变量，也就是说 OC 中类都有一个 isa 成员变量，那么要知道一个类的底层结构具体实现，可以从 isa 入手，查看 runtime 源码；

  1. 创建对象，对象的结构是由 isa 指针决定的，而 isa 指向的是 objc_class 结构体(OC的类结构)；
  2. objc_class OC 类的结构体中包含了：
    • isa_t isa，指向 isa_t 联合体类型 isa 指针(虽然不是 * 指针，但是可能底层做好了内存分配指向等，指针理解或者说isa_t类型 isa成员变量)；
    • Class superclass，每个对象结构中都包含 superclass 指针(or nil for base classes)寻找到其父类信息；
    • cache_t cache，方法缓存；
    • class_data_bits_t bits，用于获取具体的类信息；
  3. 对象被实例化后通过 isa 指针找到其所属类，而类结构中有包含 isa 指针，因而说 OC 中类本身也是一个对象 - 类对象；
