真机调试 Bug 汇总

Posted on 2016-05-01 | Post modified: 2018-07-19 | In iOS - 开发日常

真机运行提示

xxx’s iPhone is busy:Preparing debugger support for xxx’s iPhone

Xcode will continue when xxx’s iPhone is finished.

解决办法：

  1. 重启大法：重启手机及 Xcode；
  2. 专业方法：Xcode 菜单栏 Window -> 选中 Devices and Simulators -> 在继续看下图

模拟器运行提示

Failed to initiate service connection to simulator

Error returned in reply:Connection invalid

原因： 系统中安装了多个版本的 Xcode 导致；

解决办法： 清理一下 Xcode 的 Derived Data 数据，方法有两种：

  1. 命令行清理

  1. 手动清理

    Xcode 菜单栏点击 Xcode -> 选择 Xcode 偏好设置 preferences > 选中 Location > DerivedData > 选中DerivedData下方的灰色箭头 > 跳转到 DerivedData文件夹目录 > 将DerivedData文件夹删除 > 重启Xcod

真机调试运行弹框提示如下错误：

解决方法： 检查证书配置文件生产环境和开发环境是否对应正确，并且 Scheme -> Edit Scheme -> Build Configuration 里面选择Debug；

真机调试运行弹框提示如下错误：

The file “XXX.app” couldn’t be opened because you don’t have permission to view it.

解决方法：info文件中的字段Executable file 与 build settings栏中的Packaging中的Product Name 不一致就会导致的，将info.plist的文件中的Executable.file中的文件修改为:$(PRODUCT_NAME)。

Error returned in reply: Connection invalid

原因：电脑安装了多个版本的 Xcode 导致的。

解决方法：

  1. 在强制退出中关闭xcode的所有运行程序
  2. 清理DerivedData执行命令 rm -rf ~/Library/Developer/Xcode/DerivedData
  3. 重新打开工程
